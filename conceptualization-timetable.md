## 1. Business purpose

대학교 수강신청 기간마다 학생들은 원하는 과목, 공강 시간, 선호 교수님 등 여러 조건을 고려하여 시간표를 직접 짜야 한다. 그러나 수백 개의 강의 중에서 조건에 맞는 시간표를 수작업으로 찾는 것은 매우 번거롭고 시간이 많이 소요된다. 특히 시간이 겹치는 강의를 피하면서 공강 조건까지 맞추려면 경우의 수가 너무 많아 최적의 시간표를 찾기 어렵다.

이러한 문제를 해결하기 위해 학생이 원하는 과목, 희망 학점, 공강 시간대, 선호 교수님을 입력하면 우선순위 알고리즘을 통해 최적의 시간표를 자동으로 추천해주는 웹 사이트 **"스마트 시간표"** 를 개발하게 되었다.

이 프로젝트의 첫 번째 목표는 **효율적인 시간표 추천**이다. 학생이 설정한 조건들을 분석하여 시간 충돌 없이 가장 우선순위가 높은 시간표 조합을 자동으로 계산하고 제시한다. 두 번째 목표는 **사용자 맞춤형 경험 제공**이다. 단순히 시간표를 나열하는 것이 아니라 공강 선호도, 교수 선호도, 학점 목표를 종합적으로 반영한 우선도 점수를 산출하여 상위 시간표를 추천한다.

주요 타겟은 매 학기 수강신청을 앞둔 대학교 재학생이며, 나아가 복수전공·부전공 이수 계획이 복잡한 학생들에게도 유용하게 활용될 수 있다.

## 2. System context diagram

```
                        <<actor>>
                         Student
                            |
          Login, InputPreference, ViewTimetable,
          SaveTimetable, ModifyPreference
                            |
                        +--------+
  <<actor>>              |       |              <<actor>>
  CourseDB  ----------- | System | -----------  Server
  (학교 강의 DB)         |       |         saveTimetable,
                        +--------+         loadTimetable
                            |
                    generateTimetable,
                    calculatePriority,
                    checkConflict
                            |
                        <<actor>>
                         Admin
              manageCourseData, updateCourseDB
```

- Login : 로그인
- InputPreference : 선호 조건 입력 (과목, 학점, 공강, 교수)
- ViewTimetable : 추천 시간표 조회
- SaveTimetable : 시간표 저장
- ModifyPreference : 선호 조건 수정
- generateTimetable : 시간표 생성
- calculatePriority : 우선도 계산
- checkConflict : 시간 충돌 검사
- manageCourseData : 강의 데이터 관리
- updateCourseDB : 강의 DB 업데이트

## 4. Concept of operation

### 1). Register / Log-in

| Purpose  | 등록된 사용자인지 확인하고 개인화된 서비스를 제공한다.                                                                                      |
|----------|---------------------------------------------------------------------------------------------------------------------------------------------|
| Approach | 학생은 학번과 비밀번호로 회원가입을 한다. 로그인 시 입력한 학번과 비밀번호를 서버에서 조회하여 일치하면 로그인에 성공한다. 틀린 경우 오류 메시지를 출력한다. 학번은 중복이 불가능하며 ID로 사용된다. |
| Dynamics | 시스템의 기능을 사용하기 위해 로그인이 필요한 경우.                                                                                         |
| Goals    | 로그인한 학생에게 개인화된 시간표 추천 기능을 제공한다.                                                                                     |

### 2). Input preference

| Purpose  | 학생의 수강 조건을 수집하여 시간표 생성의 기준으로 삼는다.                                                                                                                   |
|----------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Approach | 학생은 수강 희망 과목 목록, 목표 총 학점, 공강을 원하는 요일 및 시간대, 선호하는 교수님 이름을 입력 폼에 작성한다. 각 항목은 우선도 계산에 가중치로 반영된다. 필수 입력 항목과 선택 입력 항목을 구분한다. |
| Dynamics | 학생이 새 학기 시간표를 구성하고자 하는 경우.                                                                                                                                |
| Goals    | 학생의 조건을 정확히 수집하여 최적의 시간표 추천 결과를 도출한다.                                                                                                            |

### 3). Generate timetable

| Purpose  | 입력된 조건을 기반으로 가능한 시간표 조합을 생성하고 우선도를 계산한다.                                                                                                                                                  |
|----------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Approach | 시스템은 강의 DB에서 학생이 선택한 과목들의 분반 정보를 가져온다. 시간 충돌이 없는 모든 조합을 탐색한다. 각 조합에 대해 공강 조건 충족 여부, 선호 교수 포함 여부, 목표 학점 달성 여부를 기준으로 우선도 점수를 산출한다. 점수 순으로 정렬하여 상위 시간표를 반환한다. |
| Dynamics | 학생이 조건 입력을 완료하고 시간표 생성을 요청한 경우.                                                                                                                                                                   |
| Goals    | 학생의 조건을 최대한 만족하는 최적의 시간표 조합을 자동으로 도출한다.                                                                                                                                                    |

### 4). View recommended timetable

| Purpose  | 우선도 순으로 정렬된 추천 시간표를 학생에게 시각적으로 제공한다.                                                                                    |
|----------|----------------------------------------------------------------------------------------------------------------------------------------------------|
| Approach | 생성된 시간표를 우선도 점수 순으로 나열하여 보여준다. 각 시간표는 주간 캘린더 형식으로 시각화된다. 학생은 시간표를 클릭하여 상세 정보(교수명, 강의실, 학점 등)를 확인할 수 있다. |
| Dynamics | 시간표 생성이 완료된 후 결과를 확인하고자 하는 경우.                                                                                               |
| Goals    | 학생이 한눈에 시간표를 비교하고 원하는 시간표를 선택할 수 있도록 한다.                                                                             |

### 5). Save timetable

| Purpose  | 학생이 선택한 시간표를 저장하여 언제든지 다시 확인할 수 있도록 한다.                                          |
|----------|--------------------------------------------------------------------------------------------------------------|
| Approach | 학생이 추천 목록에서 시간표를 선택하고 저장 버튼을 누르면 해당 시간표 정보가 서버에 저장된다. 저장된 시간표는 마이페이지에서 조회 가능하다. |
| Dynamics | 마음에 드는 시간표를 저장해두고 싶은 경우.                                                                   |
| Goals    | 학생이 수강신청 전까지 저장된 시간표를 참고할 수 있도록 한다.                                                 |

### 6). Modify preference

| Purpose  | 조건을 변경하여 새로운 시간표 추천을 받는다.                                                                                  |
|----------|------------------------------------------------------------------------------------------------------------------------------|
| Approach | 학생은 기존에 입력한 선호 조건을 수정 폼에서 변경할 수 있다. 수정 후 다시 생성 요청을 하면 새로운 조건으로 시간표가 재생성된다. |
| Dynamics | 조건을 바꿔서 다른 시간표 추천을 받고 싶은 경우.                                                                             |
| Goals    | 다양한 조건 변경을 통해 학생이 최적의 시간표를 탐색할 수 있도록 한다.                                                         |

### 7). Manage course data

| Purpose  | 매 학기 최신 강의 정보를 시스템에 반영한다.                                                                                        |
|----------|-----------------------------------------------------------------------------------------------------------------------------------|
| Approach | 관리자는 관리자 페이지에서 강의명, 담당 교수, 강의 시간, 강의실, 학점 정보를 등록·수정·삭제할 수 있다. 변경 사항은 즉시 강의 DB에 반영된다. |
| Dynamics | 새 학기 강의 편성이 확정된 경우.                                                                                                   |
| Goals    | 학생에게 항상 최신 강의 정보를 기반으로 한 시간표 추천을 제공한다.                                                                 |

## 5. Problem statement

스마트 시간표는 대학생이 매 학기 수강신청 전 최적의 시간표를 자동으로 추천받을 수 있도록 도와주는 웹 서비스이다. 이 시스템은 학생 클라이언트, 웹 서버, 강의 DB로 구성된다.

스마트 시간표의 Problem Statement는 아래와 같다.

1. 과목 수가 많아질수록 가능한 시간표 조합의 수가 기하급수적으로 증가한다. 따라서 탐색 알고리즘의 성능 최적화가 필요하며, 응답 시간이 길어지지 않도록 설계해야 한다.

2. 학교마다 강의 DB 형식이 다르기 때문에 강의 데이터를 표준화된 형식으로 가져오는 파싱 처리가 필요하다. 초기 버전에서는 관리자가 직접 데이터를 입력하는 방식으로 대체한다.

3. 우선도 계산 기준(공강 가중치, 교수 선호 가중치 등)은 학생마다 다를 수 있으므로, 가중치를 사용자가 직접 조정할 수 있는 기능을 고려해야 한다.

4. 비기능적 요구사항(NFR):
   - 성능: 시간표 생성 결과는 5초 이내에 반환되어야 한다.
   - 보안: 학생 개인정보 및 저장된 시간표는 암호화하여 보관한다.
   - 가용성: 수강신청 기간 트래픽 집중에도 서비스가 안정적으로 운영되어야 한다.
   - 사용성: 별도의 설명 없이도 직관적으로 조건을 입력하고 결과를 확인할 수 있어야 한다.

## 6. Glossary

용어사전에 대해선 다음의 표와 같다.

| Terms        | Description                                                                                                   |
|--------------|---------------------------------------------------------------------------------------------------------------|
| 우선도 점수  | 학생이 입력한 조건(공강, 선호 교수, 목표 학점)의 충족 여부를 수치화한 점수. 점수가 높을수록 추천 순위가 높다. |
| 공강         | 수업이 없는 시간대. 학생이 원하는 요일 또는 시간대에 강의가 배치되지 않은 상태를 말한다.                      |
| 분반         | 동일한 과목이 여러 시간대에 개설된 경우 각각의 강의를 분반이라 한다.                                          |
| 시간 충돌    | 두 개 이상의 강의 시간이 겹치는 상태. 시간 충돌이 있는 조합은 시간표에서 제외된다.                            |
| 강의 DB      | 학교에서 제공하는 강의 목록, 담당 교수, 강의 시간, 강의실, 학점 정보가 저장된 데이터베이스.                   |
| 가중치       | 우선도 점수 계산 시 각 조건(공강, 교수 선호 등)에 부여되는 중요도 비율.                                       |
| 목표 학점    | 학생이 해당 학기에 수강하고자 하는 총 학점 수.                                                                |
