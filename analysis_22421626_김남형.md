## 1. Introduction

### 1) Executive Summary

대학교 수강신청 기간마다 학생들은 원하는 과목, 공강 시간, 선호 교수님 등 여러 조건을 고려하여 시간표를 직접 짜야 한다. 그러나 수백 개의 강의 중에서 조건에 맞는 시간표를 수작업으로 찾는 것은 매우 번거롭고 시간이 많이 소요된다. 특히 시간이 겹치는 강의를 피하면서 공강 조건까지 맞추려면 경우의 수가 너무 많아 최적의 시간표를 찾기 어렵다.

이러한 문제를 해결하기 위해 학생이 원하는 과목, 희망 학점, 공강 시간대, 선호 교수님을 입력하면 우선순위 알고리즘을 통해 최적의 시간표를 자동으로 추천해주는 시스템 **"스마트 시간표"** 를 개발하게 되었다.

### 2) Business Goals

스마트 시간표 시스템의 첫 번째 목표는 **효율적인 시간표 추천**이다. 학생이 설정한 조건들을 분석하여 시간 충돌 없이 가장 우선순위가 높은 시간표 조합을 자동으로 계산하고 제시한다. 두 번째 목표는 **사용자 맞춤형 경험 제공**이다. 단순히 시간표를 나열하는 것이 아니라 공강 선호도, 교수 선호도, 학점 목표를 종합적으로 반영한 우선도 점수를 산출하여 상위 시간표를 추천한다.

주요 타겟은 매 학기 수강신청을 앞둔 대학교 재학생이며, 나아가 복수전공·부전공 이수 계획이 복잡한 학생들에게도 유용하게 활용될 수 있다.

### 3) Technical Goals

시스템은 크게 Client(터미널/웹 브라우저), Application Server, Course DB로 구성된다. Application Server는 학생 계정 정보, 저장된 시간표, 선호 조건 등을 관리한다. Course DB는 학교의 강의 목록·담당 교수·강의 시간·학점 정보를 저장하며 관리자가 매 학기 갱신한다.

시간표 생성 알고리즘은 학생이 선택한 과목들의 모든 분반 조합을 탐색하고, 시간 충돌이 없는 조합에 대해 우선도 점수를 산출하여 상위 결과를 반환한다. 과목 수 증가에 따른 조합 폭발 문제를 방지하기 위해 탐색 알고리즘의 성능 최적화가 필요하며, 시간표 생성 결과는 **5초 이내**에 반환되도록 설계한다.

## 3. Domain analysis

아래는 Domain Analysis에서 나오는 Class들의 관계를 간단하게 나타낸 것이다.

---

**1) Student**

스마트 시간표 시스템에서 시간표 추천을 요청하는 학생의 정보를 가지는 클래스이다.
ex) id, password, name...

---

**2) Admin**

스마트 시간표 시스템에서 강의 데이터를 관리하는 관리자의 정보를 가지는 클래스이다.
ex) id, password, manageCourseDB()...

---

**3) Preference**

학생이 시간표 생성을 위해 입력한 조건들을 저장하는 클래스이다. 각 조건은 우선도 점수 계산 시 가중치로 반영된다.
ex) 수강 희망 과목 목록, 목표 학점, 공강 요일·시간대, 선호 교수, 가중치 설정...

---

**4) TimetableGenerator**

Preference를 입력받아 CourseDB에서 분반 정보를 조회하고, 시간 충돌 검사·조합 생성·우선도 점수 계산을 수행하여 Timetable 객체 목록을 반환하는 클래스이다.
ex) generate(), checkConflict(), calcPriority(), sortByPriority()...

---

**5) Timetable**

TimetableGenerator가 생성한 하나의 시간표 조합을 나타내는 클래스이다. 우선도 점수가 높은 순으로 정렬되어 학생에게 추천된다.
ex) 포함 강의 목록, 우선도 점수, 총 학점...

---

**6) Course**

CourseDB에 등록된 개별 강의의 정보를 가지는 클래스이다.
ex) 강의명, 담당 교수, 요일, 시간, 강의실, 학점, 분반 번호...

---

**7) CourseDB**

학교에서 개설된 전체 강의 목록을 관리하는 클래스이다. Admin이 매 학기 데이터를 등록·수정·삭제하며, TimetableGenerator가 조회하여 사용한다.
ex) 강의 목록, addCourse(), editCourse(), deleteCourse(), getCourses()...

---

## 5. Glossary

용어사전에 대해선 다음의 표와 같다.

| Terms | Description |
|---|---|
| **우선도 점수** | 학생이 입력한 조건(공강, 선호 교수, 목표 학점)의 충족 여부를 수치화한 점수. 점수가 높을수록 추천 순위가 높다. |
| **공강** | 수업이 없는 시간대. 학생이 원하는 요일 또는 시간대에 강의가 배치되지 않은 상태를 말한다. |
| **분반** | 동일한 과목이 여러 시간대에 개설된 경우 각각의 강의를 분반이라 한다. |
| **시간 충돌** | 두 개 이상의 강의 시간이 겹치는 상태. 시간 충돌이 있는 조합은 시간표에서 제외된다. |
| **강의 DB (Course DB)** | 학교에서 제공하는 강의 목록, 담당 교수, 강의 시간, 강의실, 학점 정보가 저장된 데이터베이스. |
| **가중치** | 우선도 점수 계산 시 각 조건(공강, 교수 선호 등)에 부여되는 중요도 비율. 0.0 ~ 3.0 범위로 설정한다. |
| **목표 학점** | 학생이 해당 학기에 수강하고자 하는 총 학점 수. |
| **Home** | 로그인 후 처음 접근하는 메인 기능 화면. |
| **Attribute** | 객체지향 언어에서 멤버 필드에 해당한다. |
| **Operation** | 객체지향 언어에서 메서드에 해당한다. |
